# MTL Operating Doctrine — Addendum to Doctrine v1.1

**Path:** `openclaw-ops/ops/MTL.md` (canonical)  
**Dashboard:** `openclaw-ops/ops/DASHBOARD.md` (generated)  
**Delta log:** `openclaw-ops/ops/mtl_updates.ndjson` (append-only)

---

## Rules for All Stakeholders

### 1. Never hand-edit MTL.md or DASHBOARD.md
These files are generated by `mtl_apply_updates.py`. Any direct edits will be overwritten on the next run.

### 2. All updates go to mtl_updates.ndjson — append only
Each stakeholder appends one NDJSON event per line. Never edit or delete existing lines.

**Example event:**
```json
{"ts":"2026-02-26T10:00:00Z","actor":"GP-OPS","task_id":"A-RC-P3-001","op":"ADD","owner":"GP-OPS","status_to":"ACTIVE","priority":"HIGH","title":"Compaction Mitigation v1","proof_required":"YES"}
```

### 3. TASK_ID is required in all orders
Every task order from any GP agent must include a TASK_ID matching an entry in MTL.

If a TASK_ID is missing → **Hendrik must request one before starting work.**

### 4. Proof expectations must be captured as PROOF_EXPECTED events
When a proof is expected from Hendrik, the requester must append:
```json
{"ts":"...","actor":"Chip","task_id":"A-RC-P3-001","op":"PROOF_EXPECTED","proof_items":["commit hash","test evidence"]}
```

### 5. Hendrik runs mtl_update.sh after every DONE/BLOCKED batch
```bash
bash ~/.openclaw/workspace/openclaw-ops/scripts/ops/mtl_update.sh
```
This runs the updater, commits, and pushes MTL.md + DASHBOARD.md.

## Event Schema

| Field | Required | Values |
|-------|----------|--------|
| `ts` | YES | ISO8601 UTC |
| `actor` | YES | Hendrik, Chip, GP-OPS, GP-PLM, GP-GTM, GP-WEB |
| `task_id` | YES | string |
| `op` | YES | ADD, MOVE, UPDATE, PROOF_EXPECTED, PROOF_RECEIVED, COMMENT |
| `owner` | conditional | GP-OPS, GP-PLM, GP-GTM, GP-WEB, CHIP |
| `status_to` | on ADD/MOVE | ACTIVE, BLOCKED, WATCH, DONE |
| `priority` | optional | HIGH, MED, LOW |
| `title` | on ADD | short description |
| `proof_required` | optional | YES, NO |
| `proof_items` | optional | list of strings |
| `blocked_on` | on BLOCKED | CHIP, HENDRIK, EXTERNAL |
| `note` | optional | short text |

## Updater Details

Script: `openclaw-ops/scripts/ops/mtl_apply_updates.py`  
Inputs: `mtl_updates.ndjson`  
Outputs: `MTL.md`, `DASHBOARD.md`, `MTL.snapshot.json`, `~/.openclaw/watchdog/DASHBOARD.md` (mirror)

- Always exits 0
- Unknown fields in NDJSON are tolerated (forward-compatible)
- Unknown task_id on MOVE/UPDATE → logged as warning, no crash
- Same inputs → byte-identical outputs (deterministic)
- DONE tasks shown for 14 days in MTL, 7 days in Dashboard
